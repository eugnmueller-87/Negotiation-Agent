# Demos

Two self-contained, single-file demos. Open either `.html` in a browser — no build,
no server. **Every number in both is real output from the v0 deal engine**, not mock
data.

## `fleet.html` — negotiate a tail-spend category (1,000 suppliers at once)

The scale story. Procurement teams hand-negotiate their top (A/B) suppliers; the
long tail of small vendors rides on auto-renew and list price because attention
doesn't scale — yet in aggregate that tail is a large, unmanaged slice of spend.
This demo runs an agent negotiating the **entire tail in parallel**.

Three beats:
1. **The problem** — a spend-by-supplier chart showing the managed head and the
   ignored tail (~42% of spend across 800 small vendors).
2. **The fleet** — 1,000 supplier cells; press play and watch the category close in
   waves, with live € saved / closure rate / escalations.
3. **Both sides at once** — click any supplier to open a god's-eye split-screen
   (buyer vs supplier private utilities) and a convergence chart, proving the trade
   was smart (logrolling), not just fast.

The 1,000 negotiations are generated by [`scripts/gen_tailspend_demo.py`](../scripts/gen_tailspend_demo.py),
which runs `run_negotiation` (the real engine) on 1,000 varied suppliers and writes
[`fleet.json`](fleet.json). Regenerate with:

```bash
python scripts/gen_tailspend_demo.py > demo/fleet.json
```

## `peitho.html` — self-play negotiation, both sides visible

The mechanism story. One negotiation between a **buyer agent** and a **supplier**,
run as self-play, shown in **two mirrored panes** so you can see the information
asymmetry you never see in a real deal:

- **Left** — what the buyer sees (outgoing right, supplier incoming left) plus a live
  deal-state sidebar.
- **Right** — the same transcript, roles flipped, with the buyer's target and
  walk-away **redacted** (the supplier can't see the floor).
- **Reasoning drawer** under each buyer message — the actual deterministic engine
  decision: `round · threshold · incoming utility vs threshold → ACCEPT/COUNTER/ESCALATE`,
  the approved-number allowlist, and a **numeric-guard verdict** proving the drafted
  message contains no figure the engine didn't approve.
- **Meeting minutes** — at close, generate a formal, filable procurement record
  (round-by-round, agreed terms, concessions, outcome + rationale) as Markdown.

The engine, the numeric guard, and the minutes are all deterministic — a faithful JS
port of the Python core. The message prose is written by Claude when the demo runs on
claude.ai (`window.claude`), and falls back to persona-scripted templates elsewhere so
the demo always runs. Either way the engine and guard run for real.

## The one line that matters

In both demos, the **deterministic engine decides** every accept / counter / walk-away,
and an LLM only advises at the edges — it can never invent a concession. That's the
whole product thesis, made visible.
